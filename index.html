Bitcoin Public Key to Receiving Address Calculator


This is a calculator to generate a legacy, compatibility, or segwit receiving address from a compressed bitcoin child public key. One use for this is to demonstrate 'likely' ownership during an audit in which you would rather not provide your private keys, extended public key, nor a message signed by the private key.

This tool is not intended to generate multiple addresses from private keys nor parent public keys. You'll mostly find this useful if you already have a receiving address and the associated public key, or if you have a public key and want to verify it can derive a specific receiving address.

If you want to just test the calculator out, then you can also generate a random compressed WIF private key along with the derived public key and legacy receiving address. Only use these to test the calculator - do not use them to store your bitcoin!

This site was purposely designed without the option to input a private key or extended public key (as it's a security risk). If you're concerned about entering a child public key on this page, then you can download the site from Github and run it offline.
